# Weekly Study Plan

Three pacing options. All assume you **do the code and the projects**, not just read.

- **Standard (16 weeks)** — ~6–8 hrs/week. The recommended pace.
- **Accelerated (10 weeks)** — ~12–15 hrs/week. For full-time learners.
- **Weekend (24 weeks)** — ~3–4 hrs/week. Sustainable alongside a job.

## Weekend track — 24 weeks

One module every ~1.5 weeks, projects on weekends. Same order as Standard; halve the weekly load. Protect the spine weeks (03, 06, 11, 12) — give them the full time.

---

## A repeatable weekly loop

```text
Mon–Tue   Read the module (🟢→🟡→🔴), run the beginner & intermediate code
Wed       Read the 🔴 tiers + internals notes again
Thu       Build the module project (no copy-paste)
Fri       Answer interview questions out loud; note gaps
Sat       AI workflow: regenerate part of the project, review against best-practices
Sun       Rest or catch up. Spaced review of last 2 weeks' takeaways.
```

---

## Spaced repetition

Every 4th week, **don't learn new material** — instead:
- Re-answer interview questions from weeks N-4…N.
- Rebuild one small project from memory.
- Skim the cheat sheets ([`deliverables/cheat-sheets/`](../deliverables/cheat-sheets/)).

Retrieval is what moves knowledge from "I read it" to "I can do it in an interview / on the job."
